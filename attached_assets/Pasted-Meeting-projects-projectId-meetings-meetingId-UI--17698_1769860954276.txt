Meeting 상세 페이지(/projects/:projectId/meetings/:meetingId) UI를 아래 요구사항대로 수정해줘.

========================
1) Decisions 카드 UX 변경
========================
목표: ‘통합 선택’ 체크박스는 “해당 카드 전체를 통합 대상으로 선택/해제”하는 토글이다.
개별 항목 선택 UI(좌측 체크박스)는 완전히 제거한다.

요구사항:
- Decisions 카드 상단 바(header) 우측에 '통합 선택' 체크박스를 유지하되,
  체크 ON = Decisions 카드의 모든 항목이 통합 대상으로 선택됨
  체크 OFF = 통합 대상 선택 안 됨
- 체크 ON/OFF 시 항목 좌측에 체크박스가 나타나면 안 된다. (항목 좌측 체크박스는 항상 없음)
- 통합 실행 버튼(프로젝트 기억에 통합)은 ‘통합 선택’이 ON일 때만 활성화

삭제(X) UX:
- 각 항목 우측에 회색 X 아이콘을 “항상 노출” (사각 배경 없이, 연한 회색)
- hover 시:
  - 해당 항목 row의 우측 영역에만 현재처럼 빨간 배경(또는 연한 red tint)과 X 강조 표시
  - 기본 상태에서는 빨간 박스 배경이 보이지 않음
- X 클릭 시 항목 삭제(기본: 즉시 삭제 + 3초 Undo 가능하면 적용)

항목 추가:
- Decisions 카드 하단에 인라인 입력 영역을 추가:
  - placeholder: “결정 항목 추가…”
  - Enter로 추가
  - blur 시 내용이 있으면 추가
  - ESC로 입력 취소
- 추가된 항목은 즉시 리스트에 반영(optimistic)

========================
2) 다음 할 일 카드 UX 변경
========================
목표: 다음 할 일에는 체크박스가 2개 중 “완료 체크”만 남긴다.
통합 선택은 상단 바의 ‘통합 선택’ 토글로만 한다.

요구사항:
- 항목의 ‘통합용 체크박스(가장 좌측)’는 완전히 제거
- 항목의 ‘완료 체크박스(좌측에서 두번째)’는 유지:
  - 체크됨 = 완료
  - 미체크 = 미완료
  - 미완료 항목만 Overview의 Action Items에 노출 유지
- 상단 바(header) 우측에 '통합 선택' 체크박스를 추가/유지:
  - 체크 ON = 다음 할 일 카드의 모든 항목이 통합 대상으로 선택됨
  - 체크 OFF = 통합 대상 선택 안 됨
- 체크 ON/OFF 시 항목 좌측에 통합용 체크박스가 나타나면 안 됨 (항상 없음)

삭제(X) UX:
- Decisions와 동일:
  - 회색 X 아이콘 상시 노출(배경 없음)
  - hover 시에만 빨간 tint/강조

항목 추가:
- 다음 할 일 카드 하단에 인라인 입력:
  - placeholder: “할 일 추가…”
  - Enter로 추가
  - (가능하면) 추가 시 기본 상태는 미완료(체크 해제)

========================
3) Evidence Drops: 리서치 실행 플로우로 변경
========================
현재 Evidence Drops에 리서치 결과를 즉시 보여주지 말고,
초기 상태는 “논리 확인을 위해 관련 내용을 리서치 하시겠습니까?” 문구 + [실행] 버튼만 노출한다.

상태 머신:
- idle(초기):
  - 텍스트: “논리 확인을 위해 관련 내용을 리서치 하시겠습니까?”
  - 버튼: [실행]
- loading(리서치 중):
  - 텍스트: “리서치중” + 점 애니메이션(0→1→2→3→0 반복)
    예: “리서치중”, “리서치중.”, “리서치중..”, “리서치중...”
  - 실행 버튼 비활성화
- done(완료):
  - 리서치 결과 리스트 렌더링 (제목/요약/출처 링크 등)
- error(실패):
  - “리서치에 실패했습니다. 다시 시도해주세요.” + [다시 실행]

리서치 트리거:
- [실행] 클릭 시 백엔드 API 호출
- 호출 payload에는 현재 선택된 Logic Finding(또는 첫번째)을 포함
- 결과가 오면 done 상태로 전환하여 화면에 표시

주의:
- 로딩 애니메이션은 setInterval로 점 개수 순환 구현(성능 가볍게)
- 결과가 없으면 “관련 근거를 찾지 못했습니다” 안내

완료 기준:
- Decisions/다음 할 일: 개별 항목 좌측 체크박스가 절대 보이지 않는다
- 상단 바의 ‘통합 선택’만으로 카드 전체 통합 선택이 된다
- X는 회색 아이콘으로 상시 노출, hover 시에만 빨간 강조가 뜬다
- 각 카드에서 사용자가 항목을 바로 추가할 수 있다
- Evidence Drops는 실행 버튼 → 로딩 텍스트(점 애니메이션) → 결과 표시 흐름을 갖는다
